## v2.4（2026-06-17）

基于用户反馈：报告不能只是 tables and bullets，要像分析报告一样可读。

| # | 改进项 | 修改位置 | 影响 |
|:--|--------|:----|:----:|
| 1 | **Phase 5.0 写作风格指南**：叙事驱动、表格约束、子弹克制 | Phase 5 | 🔴 核心 |
| 2 | **禁止模式清单**：连续表格无过渡 / 章节开头即表格 / 数据孤岛 / 裸数据段落 | Phase 5.0 | 🔴 质量 |
| 3 | **每章结构模板**：叙述开段 → 展开分析 → 支撑表格 → 过渡句 | Phase 5.0 | 🔴 流程 |
| 4 | **表格约束**：≤6列、≤10行、表间必有叙述、标题必须分析性 | Phase 5.0 | 🟡 可读 |
| 5 | **子弹约束**：必须引入句、完整句子、长列表分组 | Phase 5.0 | 🟡 可读 |
| 6 | **执行摘要独立可读**：不依赖表格，所有数据用文字叙述 | Phase 5.0 | 🟡 可读 |
| 7 | **黄金比例**：50%叙述 / 25%图表 / 25%子弹 — 列表+表格 > 叙述 则章节不合格 | Phase 5.0 | 🔴 标准 |

**核心逻辑**：以「LLM Token Economy」报告为例 — 原报告大量使用表格逐行对比例举，读者很难抓住主线。改进后要求每章先用 2-4 句叙述开段，数据嵌入分析段落中而非孤立展示，表格仅用于需要精确对比多维度时且必须有前后解释段落。执行摘要成为可独立阅读的 mini 报告。

基于用户反馈：预提纲侦查不能只搜不爬，摘要 ≠ 内容。

| # | 改进项 | 修改位置 | 影响 |
|:--|--------|:----|:----:|
| 1 | **Step 0.2.5a2：锚定 URL 快速爬取** | Phase 0.2.5 | 🔴 核心 |
| 2 | **简报格式改为「搜索快照 + 爬取洞察」双层** | Phase 0.2.5b | 🔴 流程 |
| 3 | **Step 0.3 引用爬取的锚定源而非仅摘要** | Phase 0.3 | 🔴 流程 |
| 4 | **爬取数量按策略分级**（A:2个 / B:3-4 / C:4-5）| Phase 0.2.5a2 | 🟢 增强 |
| 5 | **新增「信息盲区」标记**（爬取未覆盖的方向）| Phase 0.2.5b | 🟡 质量 |

**核心逻辑**：搜索结果摘要常与实际内容南辕北辙（如摘要声称有定价对比，实际页面只是软广）。爬取 2-5 个锚定 URL 能大幅降低这种偏差，且增加的时间成本可控（每个 URL 5-15秒）。

| # | 改进项 | 竞品依据 | 位置 | 影响 |

| 1 | **Phase 1.5 大纲动态校准** | Anthropic 定期评估搜索对齐；OpenAI/Gemini mid-flight重规划 | Phase 1.5 | 🔴 核心 |
| 2 | **Phase 5.1 LLM-as-Judge 自评** | Anthropic 五维度评分；DeepHalluBench 中间错误不可见 | Phase 5.1 | 🔴 质量 |
| 3 | **Phase 5.2 引用抽查** | Anthropic CitationAgent；Perplexity grounded retrieval | Phase 5.2 | 🟡 质量 |
| 4 | **多表述搜索策略** | Perplexity 单查询21次搜索；Gemini 80+搜索 | Phase 1a | 🟢 增强 |

### 关键竞品发现

- **Anthropic Multi-Agent**：比单Agent性能高 90.2%，Token消耗 15×。Orchestrator-Worker 模式 + 双层并行化 → 复杂查询时间 -90%
- **Perplexity Sonar Pro**：引用精度 94.3%（最高），<3分钟完成。Grounded retrieval first 设计
- **Gemini 3 Deep Think**：HLE 48.4%（最高），1M上下文，80+搜索/查询
- **引用幻觉率**：最优系统仍有 6-37% 幻觉率；66% 为完全虚构引用
- **我们独有优势**：预提纲侦查、浏览器降级、双层停止、中英双语 — 竞品均未同时具备
